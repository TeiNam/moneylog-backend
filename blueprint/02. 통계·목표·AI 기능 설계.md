### 2-2. 목표 연동 방식

- 월별 시트의 "이번달 목표" 영역과 동일한 개념
- 지출 입력 시 한도 초과 경고 알림
- 대시보드에 목표 달성률 실시간 표시
- 주간/월간 리포트에 목표 대비 실적 자동 비교
- AI가 목표 달성 가능성 예측 및 조언

### 3-3. Bedrock 연동 아키텍처

```
[사용자 입력 (텍스트/이미지)]
         │
         ▼
[API Gateway + Lambda]
         │
         ▼
[Amazon Bedrock]
  - Claude (자연어 이해/분석)
  - 프롬프트 엔지니어링:
    · 카테고리 분류 체계 주입
    · 사용자 결제수단 목록 주입
    · 현재 예산/목표 컨텍스트 주입
         │
         ▼
[구조화된 JSON 응답]
  {
    "영역": "일반",
    "대분류": "식비",
    "소분류": "외식/배달",
    "금액": 12000,
    "결제수단": "삼성신용",
    "내용": "점심 김치찌개"
  }
         │
         ▼
[DynamoDB 저장] → [앱 UI 반영]
```

### 3-4. AI 프롬프트 설계 핵심

- 사용자의 카테고리 체계(항목 시트 기반)를 시스템 프롬프트에 포함
- 결제수단 목록을 컨텍스트로 제공
- "주유", "톨비" 등 키워드 → 자동으로 차계부 영역 분류
- "넷플릭스", "유튜브" 등 → 자동으로 구독관리 영역 분류
- "결혼식", "장례식", "축의금" 등 → 자동으로 경조사 영역 분류
- 모호한 경우 사용자에게 확인 질문
