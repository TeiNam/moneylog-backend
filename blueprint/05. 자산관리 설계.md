# 🏦 자산관리 설계

## 1. 핵심 개념

기존 `payment_method` 테이블을 `asset`(자산) 테이블로 확장.
자산 = 통장, 카드, 현금 등 돈이 오가는 모든 수단.

**소유 구조:**
- 모든 자산은 개인 소유(`user_id`) 또는 공용(`family_group_id`)
- 가족 그룹에서도 기본은 개인 자산, 필요 시 공용 자산 등록
- 거래 기록 시 본인 자산 + 공용 자산 중 선택 가능

```
자산 소유 구조
├── 👤 남편 (개인)
│   ├── 🏦 국민은행 급여통장
│   ├── 💳 삼성 신용카드
│   ├── 💳 토스 체크카드
│   └── 💵 현금
├── 👤 아내 (개인)
│   ├── 🏦 신한은행 급여통장
│   ├── 💳 국민 신용카드
│   ├── 💳 카카오 체크카드
│   └── 💵 현금
└── 👨‍👩‍👧 공용
    ├── 🏦 우리은행 생활비통장
    └── 💳 공용 체크카드
```

## 2. 데이터 모델

### 2-1. 자산 (asset) — 기존 payment_method 대체

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| user_id | UUID | FK → user (소유자, 공용이면 등록자) |
| family_group_id | UUID | FK → family_group (nullable) |
| ownership | VARCHAR | PERSONAL / SHARED |
| name | VARCHAR | 이름 (국민은행 급여통장, 삼성신용 등) |
| asset_type | VARCHAR | BANK_ACCOUNT / CREDIT_CARD / DEBIT_CARD / CASH / INVESTMENT / OTHER |
| institution | VARCHAR | 금융기관명 (국민은행, 삼성카드 등, nullable) |
| balance | BIGINT | 현재 잔액/한도 (nullable, 수동 입력) |
| memo | TEXT | 메모 (nullable) |
| icon | VARCHAR | 아이콘 (nullable) |
| color | VARCHAR(7) | 색상 코드 (nullable) |
| is_active | BOOLEAN | 활성 여부 |
| sort_order | INTEGER | 정렬 순서 |
| created_at | TIMESTAMPTZ | 생성일시 |
| updated_at | TIMESTAMPTZ | 수정일시 |

**인덱스:**
- `idx_asset_user` → (user_id)
- `idx_asset_family` → (family_group_id)

**기존 테이블 변경:**
- `transaction.payment_method_id` → `transaction.asset_id` (FK → asset)
- `subscription.payment_method_id` → `subscription.asset_id` (FK → asset)
- `user.default_payment_method_id` → `user.default_asset_id` (FK → asset)

### 2-2. asset_type 상세

| asset_type | 설명 | 잔액 의미 |
|------------|------|-----------|
| BANK_ACCOUNT | 은행 통장 (입출금, 저축, 적금 등) | 현재 잔액 |
| CREDIT_CARD | 신용카드 | 이번달 사용액 (자동 집계) |
| DEBIT_CARD | 체크카드 | 연결 통장 잔액 (수동) |
| CASH | 현금 | 보유 현금 |
| INVESTMENT | 투자 계좌 (증권, 코인 등) | 평가 금액 |
| OTHER | 기타 | 자유 입력 |

### 2-3. ownership 동작

| ownership | 조건 | 누가 볼 수 있나 | 누가 사용 가능 |
|-----------|------|:---:|:---:|
| PERSONAL | user_id만 설정 | 본인 + 가족 구성원(가족모드) | 본인만 |
| SHARED | family_group_id 설정 | 가족 구성원 전체 | 가족 구성원 전체 |


## 5. 카드 사용액 자동 집계

신용카드의 "이번달 사용액"은 거래 내역에서 자동 계산:

```sql
-- 특정 자산(카드)의 월간 사용액
SELECT SUM(t.actual_amount) AS monthly_usage
FROM transaction t
WHERE t.asset_id = :card_asset_id
  AND t.date BETWEEN :month_start AND :month_end
  AND t.type = 'EXPENSE';

-- 가족 그룹 내 구성원별 + 자산별 월간 집계
SELECT
  u.nickname,
  a.name AS asset_name,
  a.asset_type,
  a.ownership,
  SUM(t.actual_amount) AS total_amount,
  COUNT(*) AS tx_count
FROM transaction t
JOIN "user" u ON t.user_id = u.id
JOIN asset a ON t.asset_id = a.id
WHERE t.family_group_id = :group_id
  AND t.date BETWEEN :start AND :end
  AND t.type = 'EXPENSE'
GROUP BY u.id, u.nickname, a.id, a.name, a.asset_type, a.ownership
ORDER BY u.nickname, a.ownership, total_amount DESC;
```

## 6. 기존 테이블 변경 요약

| 변경 대상 | 변경 내용 |
|-----------|-----------|
| `payment_method` 테이블 | `asset` 테이블로 대체 (확장) |
| `transaction.payment_method_id` | → `transaction.asset_id` |
| `subscription.payment_method_id` | → `subscription.asset_id` |
| `user.default_payment_method_id` | → `user.default_asset_id` |
| API `settlement.py` | asset 기반 정산 로직으로 변경 |
| API 신규 | `assets.py` 라우터 추가 |

> 03. 데이터 모델 문서의 `payment_method` 관련 내용은 이 문서의 `asset` 테이블로 대체됨
