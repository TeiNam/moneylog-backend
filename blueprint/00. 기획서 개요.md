# 💰 가계부 앱 기획서

## 1. 프로젝트 개요

### 1-1. 프로젝트명
**MoneyLog** (가칭)

### 1-2. 목적
엑셀 기반 가계부의 체계적인 분류 체계와 분석 기능을 모바일/웹 앱으로 전환하여, AI(Amazon Bedrock) 기반 자동 입력 및 분석 기능을 결합한 차세대 개인 재무관리 앱 개발

### 1-3. 핵심 차별점
- **4대 영역 탭 구조**: 일반(통합) / 차계부 / 구독관리 / 경조사 — 최상단 탭으로 빠른 전환
- **통합 뷰**: 차계부·구독관리·경조사에 입력한 내역이 일반 영역에 자동 통합 표시
- **가족 그룹**: 가족 구성원별 작성자 표기, 개인별 카드 사용 현황 및 통합 정산
- **AI 채팅 입력**: Bedrock 연동 채팅창에서 자연어로 입력하면 자동 분류·기록
- **AI 분석**: 목표 대비 분석, 소비 패턴 인사이트, 절약 제안 등

### 1-4. 타겟 사용자
- 체계적 가계부 관리를 원하지만 엑셀은 불편한 사용자
- 차량 유지비, 구독 서비스, 경조사비를 별도 추적하고 싶은 사용자
- 가족 공동 가계부 + 개인별 카드 정산이 필요한 사용자
- AI를 통한 간편 입력과 재무 분석을 원하는 사용자

## 2. 앱 구조 (IA: Information Architecture)

```
┌─────────────────────────────────────────────────┐
│  [일반]    [차계부]    [구독관리]    [경조사]     │  ← 최상단 탭
├─────────────────────────────────────────────────┤
│                                                   │
│  각 영역별 콘텐츠                                  │
│                                                   │
├─────────────────────────────────────────────────┤
│  [홈] [입력] [AI채팅] [통계] [설정]               │  ← 하단 네비게이션
└─────────────────────────────────────────────────┘
```

### 2-1. 최상단 탭 (4대 영역)

| 탭 | 역할 | 비고 |
|---|---|---|
| 일반 | 전체 수입/지출 통합 뷰 | 다른 3개 영역 데이터 자동 합산 |
| 차계부 | 차량 관련 지출 전용 | 주유, 정비, 보험, 세금, 톨비 등 |
| 구독관리 | 디지털 정기구독 관리 | OTT, 앱, SaaS 등 월/연 구독 |
| 경조사 | 경조사 수입/지출 관리 | 축의금, 조의금, 선물 등 주고받은 내역 |

### 2-2. 하단 네비게이션

| 탭 | 기능 |
|---|---|
| 홈 | 현재 선택된 영역의 대시보드 |
| 입력 | 빠른 수입/지출 입력 |
| AI 채팅 | Bedrock 기반 자연어 입력 & 분석 |
| 통계 | 주간/월간/연간 통계 + 카드 정산 |
| 설정 | 카테고리 관리, 목표 설정, 가족 그룹, 계정 등 |

## 3. 데이터 흐름

```
[차계부 입력] ──┐
[구독관리 입력] ─┼──→ [PostgreSQL] ──→ [일반 영역 통합 뷰]
[경조사 입력] ──┘         │
[일반 직접 입력] ─────────┘
                          │
                          ▼
                    [통계/분석 엔진]
                          │
                          ▼
                    [AI 분석 (Bedrock)]
```

- 모든 입력은 PostgreSQL에 저장
- 각 항목에는 `영역 태그`(일반/차계부/구독/경조사) + `작성자(user_id)` 기록
- 일반 탭에서는 모든 영역의 데이터를 시간순으로 통합 표시
- 가족 모드에서는 구성원별 필터 + 카드별 정산 가능

## 4. 기술 스택 요약

| 계층 | 기술 | 비고 |
|------|------|------|
| 백엔드 | Python FastAPI | ECS Fargate 또는 EC2 배포 |
| DB | PostgreSQL (RDS) | SQLAlchemy 2.0 + Alembic |
| 웹 프론트 | React + TypeScript | Vite, Tailwind, shadcn/ui |
| iOS 앱 | Swift (SwiftUI) | 네이티브, 우선 개발 |
| Android | 추후 결정 | Kotlin 또는 React Native 검토 |
| 인증 | Cognito | 이메일 + 소셜 로그인 |
| AI | Amazon Bedrock (Claude) | 자연어 입력, 분석 |
| 파일 | S3 + CloudFront | 이미지, 웹 호스팅 |
