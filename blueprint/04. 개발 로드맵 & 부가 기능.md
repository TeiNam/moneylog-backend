# 🗓️ 개발 로드맵 & 부가 기능

## 1. 개발 단계 (MVP → 확장)

### Phase 1: MVP (핵심 기능) — 약 8~10주

| 주차 | 작업 |
|------|------|
| 1-2 | 프로젝트 셋업, DB 스키마, API 설계, Flutter 프로젝트 초기화 |
| 3-4 | 일반 영역: 수입/지출 입력, 리스트, 캘린더 뷰 |
| 5 | 차계부 영역: 전용 입력/리스트 |
| 6 | 구독관리 영역: 구독 등록/목록/알림 |
| 7 | 경조사 영역: 입력/리스트/인물별 누적 |
| 8 | 통합 대시보드, 월간 통계 기본 |
| 9-10 | 테스트, 버그 수정, UI 다듬기 |

**MVP 산출물:** 4대 영역 탭 + 입력/조회 + 기본 통계

### Phase 2: AI 연동 — 약 4~6주

| 주차 | 작업 |
|------|------|
| 1-2 | Bedrock 연동 Lambda 구축, 프롬프트 엔지니어링 |
| 3 | AI 채팅 입력 기능 (자연어 → 구조화 데이터) |
| 4 | AI 분석 기능 (지출 분석, 절약 제안) |
| 5-6 | 영수증 OCR 연동, 테스트 |

### Phase 3: 고도화 — 약 4주

| 주차 | 작업 |
|------|------|
| 1 | 목표 관리 & 예산 설정 |
| 2 | 주간/연간 통계, 차트 고도화 |
| 3 | 알림 시스템 (구독 결제일, 예산 초과, 목표 리마인드) |
| 4 | 데이터 내보내기 (엑셀 export), 설정 고도화 |

### Phase 4: 확장 (선택)

- 자산현황 관리 (계좌, 예금, 적금, 투자)
- 부채/대출 관리
- 가족 공유 가계부
- 위젯 (iOS/Android 홈화면)
- Apple Watch / WearOS 빠른 입력

## 2. 부가 기능 상세

### 2-1. 알림 시스템

| 알림 유형 | 트리거 | 내용 |
|-----------|--------|------|
| 구독 결제 예정 | 결제일 N일 전 | "넷플릭스 17,000원 결제 예정 (3/15)" |
| 예산 초과 경고 | 카테고리 예산 80% 도달 | "식비 예산 80% 소진 (480K/600K)" |
| 무지출 축하 | 하루 지출 0원 | "오늘 무지출 성공! 🎉 이번달 8일째" |
| 목표 리마인드 | 주 1회 | "저축 목표 42% 달성 중 (목표 60%)" |
| 월말 리포트 | 매월 말일 | "3월 리포트가 준비되었습니다" |
| 경조사 일정 | 등록된 일정 전일 | "내일 ○○○ 결혼식 (○○웨딩홀)" |

### 2-2. 데이터 내보내기

- 월별/연간 데이터를 엑셀(.xlsx)로 내보내기
- 기존 엑셀 가계부 양식과 호환되는 포맷 옵션
- CSV 내보내기
- PDF 리포트 생성

## 3. 핵심 사용자 시나리오

### 시나리오 1: 일상 지출 기록
1. AI 채팅탭 열기
2. "점심 김치찌개 12000원 삼성카드" 입력
3. AI가 자동 분류 → 확인 버튼 탭
4. 일반 탭 대시보드에 즉시 반영

### 시나리오 2: 주유 기록
1. 차계부 탭 → 입력
2. 주유 선택 → 금액, 리터, 주행거리 입력
3. 차계부 대시보드에 연비 자동 계산
4. 일반 탭에도 "유류비" 항목으로 통합 표시

### 시나리오 3: 구독 관리
1. 구독관리 탭 → 구독 추가
2. 넷플릭스 / 17,000원 / 매월 15일 / 삼성신용 입력
3. 매월 15일 자동으로 지출 기록 생성
4. 결제일 1일 전 알림 수신

### 시나리오 4: 경조사 기록
1. AI 채팅: "오늘 ○○○ 결혼식에서 축의금 5만원 냈어"
2. AI가 경조사 영역으로 자동 분류
3. 경조사 탭에서 ○○○ 인물 카드에 누적
4. 일반 탭에도 "경조사회비" 항목으로 통합 표시

### 시나리오 5: 월말 분석
1. AI 채팅: "이번달 분석해줘"
2. AI가 수입/지출/저축률/목표달성률 종합 분석
3. 카테고리별 전월 대비 증감 분석
4. 절약 가능 포인트 제안
5. 다음달 예산 조정 제안
